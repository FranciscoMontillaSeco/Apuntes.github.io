Gestión de bases de datos

Sistema informático: Colección de personas, procedimientos y equipos que están construidos, pegados y mantenidos con el fin de recoger, registrar, almacemar, recuperar y visualizar información.

Formado por:

  • Contenido (datos)
  • Hardware (físico)
  • Software (lógico)
  • RRHH (programadores, técnicos, etc...)

La gestión de los datos inicialmente (en un sistema informático) se hacia a través de ficheros tradicionales.

Problemas:

  • Ocupación inutil de espacio secundario en disco.
  • Aumento del tiempo de procesamiento.
  • Posibilidad de información incoherente.

A raiz de ello se hicieron las primeras bases de datos para almacenar los datos en 1 solo lugar.

Un dato se almacena en un único lugar y está disponible para todas las aplicaciónes.

Ventajas de las GBD:

  • No ocupamos más espacio.
  • Disminuimos los tiempos de procesamiento.
  • No hay posibilidad de inconsistencia/incoheréncia.
  • Existe independencia entre los datos y los procesos.
  • Los datos devuelven todas las inconcluencias.

Ventajas de las bases de datos

  1. Ventajas referidas a los datos

    • Los datos son independientes d elos tratamientos
    • Hay mejor disponibilidad de la información.
    • Mayor eficiencia en la recogida, entrada y almacenamiento del sistema

  2. Referidas a los reslultados obtenidos

    • Mayor coherencia en los datos. (1 dirección por persona en una BD)
    • Mayor valor informativo. (datos siempre correctos)
    • Mejor y más normalizada documentación de la información.

  3. Referida a los usuarios

    • Acceso mucho más rapido y sencillo.
    • Mayor facilidad para compartir los datos.
    • Mayor flexibilidad para atender demandas cambiantes.

Inconveniente del enfoque de bases de datos

  • Instalación costosa.
  • Personal especializado.
  • Implantación larga y "complicada".
  • Falta de rentabilidad a corto plazo

Estructura de un sistema de base de datos

Una base de datos es un conjunto de datos almacenados en un soporte informático, y se deben de cumplir los siguientes requisitos:

  1. Los datos deben ser estructurados y deben estár interrelacionados.
  2. No hay redundancia de la información.
  3. Hay independencia entre datos y procesos.
  4. La definición de los datos es única y está integrada en la base de datos

Esquemas de una base de datos

Este modelo se llama ansi/sparc (el de la foto que pondremos luego).

Esquema Conceptual

Se incluyen todos los datos que necesita la organización y las inter-relaciones entre ellos junto con las restricciones de integridad de los datos. Se necesitan los datos de la organización, relaciones entre estos y las relaciones de integridad referencial.Todos los datos de la base de datos deben constar en el esquema conceptual.

Esquema Externo

La visión de un usuario de una base de datos. Habrá tantas visiónes de usuarios como aplicaciones (medicos, administradores, técnicos). A cada visión que tiene un usuario se le denomina esquema externo.

Esquema Interno o Nivel Físico

En este nivel se define la estrategia de almacenamiento, los caminos de acceso (índice), la estrategia de emplazamiento de los datos, técnicas de compresión de datos, técnicas de ajuste de funcionamiento.

Conceptos y funciones de los SGBD

Para cubrir todas las necesidades de información de una organización, se requiere un software capaz de ser el interfaz entre la comunidad de usuarios y la base de datos.

SGBD

El Sistema Gestor de Bases de Datos es un conjunto de programas, procedimientos y lenguajes que suministra tanto a los usuarios informáticos como a los usuarios no informáticos los medios necesarios para describir, recuperar y manipular los datos almacenados en una base de datos garantizando su integridad, confidencialidad y seguridad.

El SGBD es el único que accederá a la base de datos, es la interfaz entre los usuarios y la base de datos.


Funciones Basicas de un SGBD

Funcion de descripción o definición

El SGBD debe definir que tipos de dato se van a almacenar en una base de datos, este debe permitir al administrador especificar los elementos que integran la base de datos, su estructura, las relaciones entre ellos, las regals de integridad, características de tipo físico y las vistas lógicas de los usuarios. Entonces, mediante un lenguaje de definición de datos (DDL) el SGBD permite al ADB (admin) que defina todos los esquemas externos, el único esquema lógico global (conceptual) y el único esquema interno.
