MODELO ENTIDAD/RELACIÓN (ER)

Estructura básica

La estructura básica del moderlo E/R es la tabla o relación. (pasar el modelo conceptual a tablas relacionadas)

Propiedades de las tablas/relaciones

  • Cada tabla tiene un nombre distinto.
  • Cada atributo de la tabla toma un solo valor en cada tupla o fila.
  • Cada atributo tiene un solo nombre.
  • Cada tupla es única. No se repiten.
  • El orden de los atributos no importa.
  • El orden de las tuplas no importa.

Tipos de claves

  • Clave candidata :
     Conjunto de atributos que identifican unidicamente cada tupla de la relacción, es decir columnas cuyos valores no se repiten en ninguna fila de la tupla.
  • Clave primaria :
    Es la candidata que se coje como identificador de las tuplas, se elije la que mejor identifique a la tupla en la base de datos.
  • Clave alternativa :
    Cualquiera que no sea primaria.
  • Clave externa :
    Son los datos de atributos de una tabla cuyos valores están relacionados con atributos de otras tablas.

Restrinciones

  • Inerentes : Son las restricciones que están siempre y no son determindas por el usuario sino por el hecho que la base de datos sea relacional.
    Las más importante son:
    • No puede haber dos tipos.
    • El orden las tuplas no es importante.
    • El orden de los atributos no importa.
    • 1 valor de atributo por tupla.
  • Semánticas : El modelo relacional permite a los usuarios incorporar restrinciones personales a los datos. Acontinuaciín se definen las semánticas:
    • Primary Key / clave principal (PK) :
      Marca uno o más atributos como identificadores de la tabla, de esta forma las filas de las tablas no podrán repetir valores, ni dejarlos vacíos. Tienen por defecto unique y Not null

    • Unicidad / Unique :
      Impide que se puedan repetir valores.

    • Not Null :
       La obligatoriedad hace que el atributo marcado deba ser incluido, es decir no puede tener valor = nulo.

    • Integridad referencial / foreign key :
       Sirve para indentificar una clave externa a la tabla, también llamada foranea, sobre uno o más atributos. Los atributos marcados de esta forma solo podran contener valores que estén relacionados con la pk de la tabla que relaciono, llamada tabla principal.
