TEMA 2 - SGBD

CONCEPTOS Y FUNCIONES

Para cubrir todas las necesidades de información de una organización se requiere un software capaz de ser el interfaz entre la comunidad y la bes de datos.
El SGBD es un conjunto coordinado de porgramas, procedimientos y lenguajes que suministra tanto a los usuarios informáticos como a los no informáticos los medios necesarios para descubrir, recuperar y manipular los datos almacenados en una base de datos, garantizando su integridad, confiencialidad y seguridad.
El SGBD es el único que accederá a la base de datos, es la interfaz entre el usuario y la base de datos y está soportado por el sistema operativo.

FUNCIONES BÁSICAS DE UN SGBD

  1. DEFINICIÓN (DDL). El SGB debe definir qué tipos de datos se van a almacenar en una base de datos, es decir, debe permitir al administrador especificar los elementos que integran la base de datos, su estructura, las relacciones entre ellos, las reglas de integridad, características de tipo físicos y vistas lógicas de los usuarios.
    Mediante el DDL el SGBD permite al administrador que defina todos los esquemas externos, el único esquema lógico global/conceptual y el único esquema interno.

  2. MANIPULACIÓN (DML). Permite buscar, añadir, suprimir o modificar los datos de la base de datos.

  3. CONTROL (DCL). Reestructurar la base de datos, obtener estadística, permisos, copias de seguridad.
  4. Es SGBD es la interfaz entre el usuario y la base de datos, es decir, el usuario hace peticiones al SGBD y es el SGBD que traduce esas peticiones al sistema opertaivo.
