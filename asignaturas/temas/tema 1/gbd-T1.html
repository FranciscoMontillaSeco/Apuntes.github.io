SISTEMAS INFORMÁTICOS

Un sistema informático puede definirse como una colección de personas, procedimientos y equipos diseñados construidos operados y mantenidos paraa recoger, registrar, almacenar, recuperar y visualizar información.

    Un sistema informático está confirmado por los siguientes componentes:
  1. Contenido (los datos)
  2. El equipo físico(hardware)
  3. El equipo lógico(software)
  4. El equipo Humano (usuarios, administradores, programadores, ...)

La gestión de los datos originalmente en un sistema informático se hacía a través de ficheros tradicionales.

    Problemas:
  1. Ocupación inútil de memoria secundario(disco).
  2. Aumento del tiempo de proceso
  3. Posibilidad de incosistencia de la información.
  4. Aparecen las primeras bases de datos, de tal modo que la información se almacena en un solo lugar y está disponible para todas las aplicaciones.
  5. Gracias a los gestores se logra evitar redundancia y problemas de inconsistencia de información entre tablas y ficheros y obtenemos capacidad de relación.
    Ventajas:
  1. No ocupamos más espacios por no tener redundancia entre ficheros.
  2. Disminuimos los tiempos de proceso.
  3. No hay posibilidad de consistencias.
  4. Existe dependencia entre los datos y los procesos.
  5. Los datos resuelven todas las expectativas que se plantean.

VENTAJAS DE ENFOQUE

    1. Referidas a los datos
    • Los datos son independientesde los tratamientos
    • Mejor disponibilidad de la información
    • Mayor eficiencia en la recogida, codificación y entrada al sistema de información.
    2. Referidas a los resultados obtenidos
    • Mayor coherencia, no hay posibilidad de consultar un dato y te devuelva dos datos distintos.
    • Mayor valor informativo, los datos siempre son correctos si me metieron correctamente.
    • Mejor y más normalizada documentación de la información
    3. Referida a los usuarios
    • Acceso mucho más rápido y sencillo
    • Mayor facilidad para compartir los datos
    • Mayor felxibilidad para atender demandas cambiantes

INCONVENIENTES DEL ENFOQUE

  1. Infraestructura costosa
  2. Personal especializado
  3. Implantación larga y complicada
  4. Falta de rentabilidad a corto plazo

ESTRUCTURA DE BASE DE DATOS

Una base de datos ens un conjunto de datos almacenados en un soporte informático y se deben de cumplir los siguientes datos almacenados en un soporte informático y se deben cumplir los siguientes requisitos:

  1. Los datos deben ser estructurados y deben tener formato (Val, Bul, Char, etc).
  2. No hay redundancia de la información, lso datos se almacenan en un único lugar.
  3. Hay idependencia entre datos y procesos
  4. La definición de los datos es única y está integrada en la base de datos.

MODELO ANSI/SPARC

  1. El esquema externo es la visión que tiene un usuario de la base de datos. Habrá tantas visiones de usuarios como aplicaciones (programas y conexión de usuario). A cada visión que tiene el usuario de la base de datos se le denomina esquema externo.
  2. En el esquema conceptual se incluyen todos los datos que necesita la organización y las interrrelaciones entre ellos, junto con las restrincciones de integridad de los datos. Un ejemplo de integridad es que no es posible dar de alta un periodo sin dar de alta previamente al cliente relacionado, esto se llama integridad referencial. Todos los datos presentes en la base de datos deben estar referenciados en el esquema conceptual.
  3. En el modelo físicoo interno se definen:
    • Estrategia de almacenamiento
    • Caminos de acceso (índices)
    • La estrategia de emplazamiento de los datos
    • Técnicas de compresión de datos
      • Técnicas de funcionamiento (tunning y paginación)
